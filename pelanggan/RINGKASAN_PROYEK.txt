 📋 FILE DESCRIPTIONS
═════════════════════════════════════════════════════════════════════════════

1. order_tambah.php (SHOPPING CART PAGE)
   Fungsi: Menampilkan semua item di keranjang
   Fitur:
   • Render cart items dari localStorage
   • Tombol +/- untuk ubah qty
   • Tombol Hapus untuk remove item
   • Real-time total calculation
   • Form metode pembayaran
   • Submit ke order_aksi.php
   
   Query: fetch produk dari get_cart_items.php

2. order_aksi.php (ORDER PROCESSING)
   Fungsi: Memproses order baru
   Fitur:
   • Ambil data dari cart
   • Validasi metode pembayaran
   • Check stock tersedia
   • Hitung total & pajak
   • Insert ke tabel orders
   • Insert ke tabel order_items
   • Deduct stock produk
   • Clear localStorage cart
   • Redirect ke invoice
   
   Database:
   • INSERT INTO orders
   • INSERT INTO order_items
   • UPDATE produk (stock)

3. order_invoice.php (INVOICE/RECEIPT)
   Fungsi: Menampilkan struk order
   Fitur:
   • Ambil data order dari database
   • Tampil info order lengkap
   • List semua item + qty + harga
   • Hitung subtotal, pajak, grand total
   • Status badge
   • Tombol cetak
   • Tombol lanjut belanja
   
   Query: SELECT orders JOIN user, SELECT order_items JOIN produk

4. kategori.php (CATEGORY FILTER)
   Fungsi: Filter produk berdasarkan kategori
   Fitur:
   • Terima parameter kategori dari URL
   • Filter produk sesuai kategori
   • Tampil product grid yang responsif
   • Navigation kategori
   • Tombol add to cart per produk
   • Redirect ke order_tambah.php
   
   Kategori support:
   • Semua
   • Kue Kering
   • Kue Basah
   • Cookies
   • Cake & Brownies
   • Snack Manis

5. get_cart_items.php (CART API)
   Fungsi: Fetch data produk untuk cart
   Method: POST (JSON)
   Input: Array product IDs
   Output: Array product data (JSON)
   
   Data returned:
   • produk_id
   • nama_produk
   • harga

6. index.php (HOMEPAGE - UPDATED)
   Update: JavaScript section untuk cart management
   Perubahan:
   • Add event listener untuk "+ Keranjang"
   • Simpan ke localStorage
   • Redirect ke order_tambah.php
   • Add event listener untuk kategori click
   • Redirect ke kategori.php dengan filter

 📊 DATABASE REQUIREMENTS
═════════════════════════════════════════════════════════════════════════════

REQUIRED TABLES:

1. user (sudah ada)
   - user_id (INT, PK)
   - nama (VARCHAR)
   - email (VARCHAR)
   - password (VARCHAR)
   - role (VARCHAR) ← 'pelanggan', 'admin', dll

2. produk (sudah ada, tapi pastikan lengkap)
   - produk_id (INT, PK)
   - nama_produk (VARCHAR)
   - harga (DECIMAL)
   - stok (INT)
   - gambar (VARCHAR)
   - kategori_id (INT, FK) ← Optional

3. kategori (sudah ada)
   - kategori_id (INT, PK)
   - nama_kategori (VARCHAR)

4. orders (sudah ada, update kolom)
   - orders_id (INT, PK, AUTO)
   - invoice (VARCHAR, UNIQUE)
   - user_id (INT, FK)
   - total (DECIMAL)
   - pajak (DECIMAL)
   - metode_pembayaran (VARCHAR)
   - status (VARCHAR) ← DEFAULT 'pending'
   - created_at (TIMESTAMP) ← ADD COLUMN jika belum ada
   - updated_at (TIMESTAMP) ← OPTIONAL

5. order_items (sudah ada)
   - order_items_id (INT, PK, AUTO)
   - orders_id (INT, FK)
   - produk_id (INT, FK)
   - qty (INT)
   - harga (DECIMAL)
   - subtotal (DECIMAL)
